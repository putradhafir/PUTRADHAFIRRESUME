1. CLASS BUKU

Bayangkan constructor sebagai "proses kelahiran" dalam siklus hidup objek. Ia adalah method istimewa yang terpanggil secara otomatis saat kata kunci new (atau secara implisit) digunakan—menandai momen pertama sebuah objek hadir di alam memori. Dalam perspektif filosofis, constructor adalah ruh yang memberi bentuk pada jasad class.

Di ekosistem Dart, constructor bukan sekadar pengisi nilai awal. Ia adalah penjaga integritas objek—memastikan setiap instance lahir dalam keadaan valid dan siap pakai. Tanpa constructor, kita harus mengatur properti satu per satu secara manual, yang rawan inkonsistensi. Constructor adalah jawaban Dart atas kebutuhan akan creational hygiene—kebersihan dalam proses penciptaan.

2. CLASS MOBIL

Constant constructor adalah jawaban Dart atas kebutuhan akan objek yang immutable—objek yang sekali diciptakan, takkan pernah berubah selamanya. Dengan menambahkan kata kunci const, kita memberi "sumpah" pada objek untuk tidak mengubah identitasnya. Semua propertinya pun harus final, seperti takdir yang sudah ditetapkan.

Dari sudut pandang komputasi, constant constructor memungkinkan canonicalization: objek-objek dengan nilai yang sama akan menempati lokasi memori yang sama, menghemat sumber daya. Ini seperti konsep tauhid dalam pemrograman—objek-objek yang serupa pada hakikatnya adalah satu entitas yang sama. 


// Nama        : Putra Dhafir Nashrullah
// NIM         : 251410009
// Kelas       : SISTEM INFORMASI SI2A

'dart:convert';
class Mobil {
  String? merk;
  int? tahun;

  // Parameterized Constructor
  Mobil(this.merk, this.tahun);
  
  // Named Constructor
  Mobil.baru() {
    merk = 'ayla';
    tahun = 2025;
  }

  // Named Constructor dariJson
  Mobil.dariJson(Map json) {
    merk = json['merk'];
    tahun = json['tahun'];
  }

  @override
  String toString() {
    return 'Mobil{merk: $merk, tahun: $tahun}';
  }
}

// Fungsi main() wajib ada untuk menjalankan program
void main() {
  // Membuat objek dengan Parameterized Constructor
  Mobil mobil1 = Mobil('toyota', 2024);
  print('Mobil 1: $mobil1');

  // Membuat objek dengan Named Constructor baru()
  Mobil mobil2 = Mobil.baru();
  print('Mobil 2: $mobil2');

  // Membuat objek dengan Named Constructor dariJson()
  Map dataMobil = {'merk': 'honda', 'tahun': 2021};
  Mobil mobil3 = Mobil.dariJson(dataMobil);
  print('Mobil 3: $mobil3');
}

3. CLASS WARNA

Initializer list adalah ruang persiapan sebelum constructor body dieksekusi—seperti fase antenatal dalam siklus hidup objek. Ditandai dengan tanda titik dua (:) setelah parameter constructor, ia memungkinkan kita memanipulasi nilai sebelum objek benar-benar "dilahirkan" ke alam memori.

Keistimewaan initializer list: ia dapat menginisialisasi field final yang bahkan belum diterima sebagai parameter. Ia juga bisa melakukan validasi awal, konversi nilai, atau perhitungan sederhana. Ini adalah bentuk pre-natal care dalam pemrograman berorientasi objek. 


// Nama        : Putra Dhafir Nashrullah
// NIM         : 251410009
// Kelas       : SISTEM INFORMASI SI2A
  class Warna {
  final int r, g, b;
  const Warna(this.r, this.g, this.b);

  @override
  String toString() {
    return 'Warna(r: $r, g: $g, b: $b)';
  }
}

void main() {
  const merah = Warna(255, 0, 0);
  const merahLagi = Warna(255, 0, 0); // Menggunakan instance yang sama di memory

  print(merah);
  print(merahLagi);
  
  // Membuktikan bahwa instance-nya sama (canonicalized)
  print(identical(merah, merahLagi)); // true
}

4. CLASS TITIK

Dart tidak mengenal kata kunci private atau public seperti bahasa lain. Sebagai gantinya, ia menggunakan konvensi yang elegan namun tegas: underscore (_). Sebuah identitas yang diawali underscore menandakan bahwa ia adalah "wilayah privat"—hanya bisa diakses dari dalam library yang sama.

Ini adalah bentuk information hiding yang fundamental dalam enkapsulasi. Seperti tembok Masjid yang memisahkan area ibadah dari area publik, underscore menjaga data sensitif agar tidak diakses sembarangan. Namun perlu diingat: privat di Dart adalah privat di level library, bukan per class—sebuah nuansa yang sering luput dari pemahaman. 


// Nama        : Putra Dhafir Nashrullah
// NIM         : 251410009
// Kelas       : SISTEM INFORMASI SI2A
  class Titik {
  final double x, y;

  Titik(double x, double y)
    : x = x.abs(),
      y = y.abs() {
    print('Titik dibuat di (${this.x}, ${this.y})');
  }
}

void main(){
  // Contoh penggunaan class Titik
  print('Membuat objek Titik pertama:');
  var titik1 = Titik(3.0, 4.0);
  print('Titik 1: x = ${titik1.x}, y = ${titik1.y}');

  print('\nMembuat objek Titik kedua dengan nilai negatif:');
  var titik2 = Titik(-5.0, 10.0);
  print('Titik 2: x = ${titik2.x}, y = ${titik2.y}');

  print('\nMembuat objek Titik ketiga dengan nilai negatif di kedua koordinat:');
  var titik3 = Titik(-1.5, -2.5);
  print('Titik 3: x = ${titik3.x}, y = ${titik3.y}');
}

5. PRAKTIK 1 CLASS MAHASISWA

Jika data privat adalah ruang tersembunyi, maka getter adalah jendela yang memungkinkan kita melihat ke dalam tanpa bisa menyentuh. Getter adalah method khusus yang memberikan akses baca (read-only) ke field privat, menjaga agar data tidak termodifikasi secara langsung.

Dalam praktiknya, getter bisa melakukan lebih dari sekadar mengembalikan nilai. Ia bisa melakukan komputasi, formatting, atau bahkan logging setiap kali data diakses. Getter adalah bentuk controlled transparency—kita bisa melihat, tapi tidak bisa mengubah tanpa melalui mekanisme yang ditentukan. 


// Nama        : Putra Dhafir Nashrullah
// NIM         : 251410009
// Kelas       : SISTEM INFORMASI SI2A
  class Mahasiswa {
  String nama;
  int umur;
  String? nim;
  
  // Constructor utama
  Mahasiswa(this.nama, this.umur);
  
  // Named constructor: mahasiswa tanpa NIM (baru daftar)
  Mahasiswa.baru(this.nama, this.umur) : nim = "Belum diisi";
  
  // Named constructor: mahasiswa anonim (untuk privasi)
  Mahasiswa.anonim(this.umur) 
      : nama = "Anonymous",
        nim = "****";
  
  // Named constructor: dari Map (untuk deserialization)
  Mahasiswa.fromJson(Map json)
      : nama = json['nama'],
        umur = json['umur'],
        nim = json['nim'];
  
  void tampilkan() {
    print("Nama: $nama, Umur: $umur, NIM: $nim");
  }
}

void main() {
  var m1 = Mahasiswa("PUTRA", 18);
  var m2 = Mahasiswa.baru("Budi", 19);
  var m3 = Mahasiswa.anonim(22);
  var m4 = Mahasiswa.fromJson({
    'nama': 'PUTRA',
    'umur': 18,
    'nim': '251410009'
  });
  
  m1.tampilkan();
  m2.tampilkan();
  m3.tampilkan();
  m4.tampilkan();
}

6. ENKAPSULASI CLASS PERSON

Jika getter adalah jendela, maka setter adalah pintu gerbang yang dijaga ketat. Setter memungkinkan perubahan data privat, namun dengan aturan yang telah ditentukan. Sebelum nilai baru disimpan, setter bisa melakukan validasi, sanitasi, atau bahkan menolak perubahan jika tidak memenuhi syarat.

Inilah esensi enkapsulasi: jangan biarkan data mentah keluar, jangan biarkan data sembarangan masuk. Setter adalah filter yang memastikan hanya data berkualitas yang boleh mengubah keadaan internal objek. Dalam konteks keamanan, setter adalah guard yang mencegah korupsi data.

7. CLASS GETTER DAN SETTER

Dalam hidup, kita bisa mencapai tujuan melalui berbagai jalan. Demikian pula dalam Dart, sebuah class bisa memiliki banyak constructor—masing-masing menawarkan cara berbeda untuk menciptakan objek. Inilah yang disebut named constructor.

Named constructor memungkinkan kita memberikan nama yang deskriptif pada setiap cara penciptaan objek. Misalnya, Mahasiswa.baru(), Mahasiswa.fromJson(), Mahasiswa.anonim(). Setiap constructor bisa memiliki logika inisialisasi yang berbeda, namun semuanya menghasilkan objek dari class yang sama. Ini adalah bentuk creational polymorphism—satu class, banyak cara lahir. 


// Nama        : Putra Dhafir Nashrullah
// NIM         : 251410009
// Kelas       : SISTEM INFORMASI SI2A
class Person{
  String?_name;
  int?age;

  Person(this._name,this.age);

  // Getter (konsistenkan nama untuk kemudahan pemakaian)
  String?get nama => _name;
  String?getName() => _name;

  // Setter (konsistenkan nama)
  set nama(String?value) => _name = value;
  void setName(String?value) => _name = value;
}

// Tambahkan fungsi main() untuk menjalankan program
void main() {
  // Membuat objek Person
  Person orang = Person("Putra", 18);
  
  // Menggunakan getter
  print("Nama awal: ${orang.nama}");
  print("Nama via getName(): ${orang.getName()}");
  
  // Menggunakan setter
  orang.nama = "Dhafir";
  print("Nama setelah diubah: ${orang.nama}");
  
  orang.setName("Nashrullah");
  print("Nama via setName(): ${orang.getName()}");
}

8. CLASS PRODUK

Setelah memahami teori, mari kita lihat enkapsulasi dalam skenario nyata: sistem login. Di sini, data sensitif seperti username dan password disembunyikan di balik field privat. Tidak ada akses langsung dari luar—semua interaksi harus melalui method publik yang sudah divalidasi.

Inilah keindahan enkapsulasi: pengguna class hanya perlu tahu apa yang bisa dilakukan (method cekLogin), tanpa perlu tahu bagaimana data disimpan (field _username, _password). Kita bisa mengubah struktur internal kapan saja—misalnya menambahkan enkripsi, hashing, atau database—tanpa mengganggu kode yang menggunakan class ini. 


// Nama  : PUTRA DHAFIR NASHRULLAH
// Nim   : 251410009
// Kelas : SI2A

class Produk {
  String _nama;
  double _harga;

  Produk(this._nama, this._harga);

  String get nama => _nama;
  double get harga => _harga;

  set nama(String value) {
    if (value.length >= 3) {
      _nama = value;
    } else {
      print('Gagal memperbarui nama: Nama harus memiliki minimal 3 karakter.');
    }
  }

  set harga(double value) {
    if (value > 0) {
      _harga = value;
    } else {
      print('Gagal memperbarui harga: Harga harus lebih besar dari 0.');
    }
  }
}

void main() {
  var produk1 = Produk('Monitor Samsung', 3500.0);

  print('--- Informasi Produk Awal ---');
  print('Nama Produk: ${produk1.nama}');
  print('Harga Produk: Rp ${produk1.harga}');

  print('\n--- Memperbarui Produk ---');
  produk1.nama = 'Keyboard Mekanik';
  produk1.harga = 1200.0;

  print('Nama (setelah update): ${produk1.nama}');

  print('\n--- Menguji Validasi Setter ---');
  produk1.nama = 'TV'; // Gagal
  print('Nama tetap: ${produk1.nama}');

  produk1.harga = -100.0; // Gagal
  print('Harga tetap: Rp ${produk1.harga}');
}

9. PRAKTIK REKENING BANK

Setiap class adalah blueprint yang mendefinisikan struktur dan perilaku objek. Seperti cetakan kue yang menentukan bentuk akhir kue, class menentukan bagaimana objek akan diciptakan dan berinteraksi. Dalam pemrograman berorientasi objek, class adalah fondasi dari segalanya.

Class membungkus data (properti) dan perilaku (method) dalam satu kesatuan yang kohesif. Ini adalah bentuk abstraksi yang memungkinkan kita memodelkan entitas dunia nyata ke dalam kode. Semakin baik desain class, semakin mudah pula kita membangun sistem yang kompleks. 


// Nama        : Putra Dhafir Nashrullah
// NIM         : 251410009
// Kelas       : SISTEM INFORMASI SI2A
  
class RekeningBank {
  String _nomorRekening;
  double _saldo;

  RekeningBank(this._nomorRekening, this._saldo);

  void deposit(double jml) {
    if (jml > 0) {
      _saldo += jml;
      print('Deposit Rp $jml Berhasil.');
    }
  }

  void tarik(double jml) {
    if (jml > 0 && _saldo >= jml) {
      _saldo -= jml;
      print('Tarik Rp $jml Berhasil.');
    } else {
      print('Gagal: Saldo tidak cukup atau input tidak valid.');
    }
  }

  void cekSaldo() {
    print('Rekening: $_nomorRekening | Saldo: Rp $_saldo');
  }
}

void main() {
  var bank = RekeningBank('2514-10002', 2000000);

  bank.cekSaldo();
  bank.tarik(500000);
  bank.cekSaldo();
}
