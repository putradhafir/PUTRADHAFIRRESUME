1. Pengenalan Sealed Class

Pengertian Sealed Class:

  • Sealed class adalah jenis class dalam Dart yang membatasi pewarisan hanya dalam satu file yang sama.
  • Pembatasan ini memungkinkan pengembang untuk mengontrol semua class turunan secara lebih jelas.
  • Dengan sealed class, program menjadi lebih aman, terorganisir, dan mudah dipahami.
  • Pada program, ApiResponse digunakan sebagai sealed class dengan dua turunan, yaitu Success dan Error.
  • Kedua class tersebut merepresentasikan kondisi berhasil dan gagal.
  • Hal ini memudahkan pengelolaan alur program serta pengecekan tipe data.

Kode Program VS Code:


// Nama  : Putra Dhafir Nashrullah
// NIM   : 251410009
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void main() {
  print("Contoh penggunaan Sealed Class");
}

3. Polimorfisme (Method Overriding)

Polimorfisme adalah konsep dalam OOP yang memungkinkan sebuah method memiliki banyak bentuk perilaku. Salah satu bentuk polimorfisme adalah method overriding, yaitu ketika subclass mendefinisikan ulang method dari superclass. Pada program, class Animal memiliki method speak() sebagai method umum. Class Dog dan Cat sebagai turunan melakukan overriding pada method tersebut untuk menghasilkan suara yang berbeda.

// Nama : PUTRA DHAFIR NASHRULLAH // NIM : 251410009 // Kelas : SI2A class Animal { void speak() { print("Hewan bersuara"); } } class Dog extends Animal { @override void speak() { print("Anjing: Woof!"); } } class Cat extends Animal { @override void speak() { print("Kucing: Meow!"); } } void main() { Animal a = Animal(); a.speak(); Animal d = Dog(); d.speak(); Animal c = Cat(); c.speak(); }

4. Abstraksi (Abstract Class)

Abstract class adalah class yang tidak bisa diinstansiasi langsung, tetapi berfungsi sebagai kerangka bagi class turunan. Method abstrak di dalamnya wajib diimplementasikan oleh subclass. Pada contoh program, abstraksi diterapkan pada class Bentuk yang memiliki method hitungLuas(), kemudian diimplementasikan oleh class Persegi dan Lingkaran dengan hasil perhitungan luas yang berbeda.

// Nama : PUTRA DHAFIR NASHRULLAH // NIM : 251410009 // Kelas : SI2A abstract class Bentuk { double hitungLuas(); } class Persegi extends Bentuk { double sisi; Persegi(this.sisi); @override double hitungLuas() { return sisi * sisi; } } class Lingkaran extends Bentuk { double jariJari; Lingkaran(this.jariJari); @override double hitungLuas() { return 3.14 * jariJari * jariJari; } } void main() { Persegi persegi = Persegi(4); print("Luas Persegi: ${persegi.hitungLuas()}"); Lingkaran lingkaran = Lingkaran(7); print("Luas Lingkaran: ${lingkaran.hitungLuas()}"); }

5. Mixin dengan State

Mixin adalah cara menambahkan perilaku ke class tanpa pewarisan (mirip multiple inheritance). Dengan keyword with, sebuah class bisa memiliki kemampuan dari banyak mixin. Mixin juga bisa memiliki state (properti). Pada contoh program, mixin CanFly dan CanSwim memiliki state _isFlying dan _isSwimming untuk mengatur kondisi objek.

// Nama : PUTRA DHAFIR NASHRULLAH // NIM : 251410009 // Kelas : SI2A mixin CanFly { bool _isFlying = false; void fly() { if (!_isFlying) { _isFlying = true; print("Started flying!"); } else { print("Already flying!"); } } void land() { if (_isFlying) { _isFlying = false; print("Landed!"); } else { print("Already on the ground!"); } } } class Bird with CanFly {} void main() { Bird bird = Bird(); bird.fly(); bird.land(); }

📝 Kesimpulan

  • Sealed class digunakan untuk membatasi pewarisan dalam satu file, cocok untuk state management.
  • Final class mencegah pewarisan total, ideal untuk utility class dan pengaturan.
  • Abstract class menjadi kerangka dasar dengan method wajib implementasi.
  • Mixin memungkinkan code reuse tanpa pewarisan tunggal.
  • Polimorfisme membuat kode lebih fleksibel dan mudah diperluas.
  • Penggunaan class modifier membuat program lebih terstruktur, aman, dan mudah di-maintain.
