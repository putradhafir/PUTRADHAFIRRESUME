🔒 Pengertian Sealed Class

Definisi Sealed class adalah jenis class dalam Dart yang membatasi pewarisan hanya dalam satu file yang sama. Pembatasan ini memungkinkan pengembang untuk mengontrol semua class turunan secara lebih jelas. Dengan sealed class, program menjadi lebih aman, terorganisir, dan mudah dipahami.

Pada contoh program, ApiResponse digunakan sebagai sealed class dengan dua turunan, yaitu Success dan Error. Kedua class tersebut merepresentasikan kondisi berhasil dan gagal. Hal ini memudahkan pengelolaan alur program serta pengecekan tipe data.

📌 Kode Program:

// Nama  : Putra Dhafir Nashrullah
// NIM   : 251410009
// Kelas : SI2A

sealed class ApiResponse {}

class Success extends ApiResponse {
  final String message;
  Success(this.message);
}

class Error extends ApiResponse {
  final String errorMessage;
  Error(this.errorMessage);
}

void handleResponse(ApiResponse response) {
  if (response is Success) {
    print("✅ Success: ${response.message}");
  } else if (response is Error) {
    print("❌ Error: ${response.errorMessage}");
  }
}

void main() {
  handleResponse(Success("Data berhasil diambil"));
  handleResponse(Error("Terjadi kesalahan jaringan"));
}

📊 Sealed Class dengan Pattern Matching

Sealed class sangat powerful jika dipadukan dengan pattern matching (switch expression) di Dart. Semua kemungkinan subclass sudah diketahui compiler, sehingga kode lebih aman dari error null atau tipe tak terduga.

sealed class Status {}
class Loading extends Status {}
class Loaded extends Status {
  final String data;
  Loaded(this.data);
}
class Failed extends Status {
  final String error;
  Failed(this.error);
}

void handleStatus(Status status) {
  switch (status) {
    case Loading(): print("⏳ Memuat...");
    case Loaded(:final data): print("✅ Data: $data");
    case Failed(:final error): print("⚠️ Error: $error");
  }
}

🔄 Polimorfisme (Method Overriding)

Konsep Polimorfisme adalah konsep dalam OOP yang memungkinkan sebuah method memiliki banyak bentuk perilaku. Salah satu bentuk polimorfisme adalah method overriding, yaitu ketika subclass mendefinisikan ulang method dari superclass.

Pada program, class Animal memiliki method speak() sebagai method umum. Class Dog dan Cat sebagai turunan melakukan overriding pada method tersebut untuk menghasilkan suara yang berbeda. Hal ini menunjukkan bahwa method yang sama dapat memiliki perilaku berbeda tergantung objek yang memanggilnya.

// Nama  : Putra Dhafir Nashrullah
// NIM   : 251410009
// Kelas : SI2A

class Animal {
  void speak() {
    print("Hewan bersuara");
  }
}

class Dog extends Animal {
  @override
  void speak() {
    print("🐕 Anjing: Woof! Woof!");
  }
}

class Cat extends Animal {
  @override
  void speak() {
    print("🐈 Kucing: Meong~");
  }
}

void main() {
  Animal hewan1 = Dog();
  Animal hewan2 = Cat();
  
  hewan1.speak(); // Output: Anjing: Woof! Woof!
  hewan2.speak(); // Output: Kucing: Meong~
}

📦 Final Class di Dart

Pembatas Pewarisan Final class adalah class yang tidak dapat diwarisi atau di-extend oleh class lain. Tujuannya adalah menjaga konsistensi struktur dan perilaku class agar tidak berubah. Cocok digunakan untuk utility class atau singleton.

// Nama  : Putra Dhafir Nashrullah
// NIM   : 251410009
// Kelas : SI2A

final class Settings {
  String theme;
  int fontSize;
  
  Settings({required this.theme, required this.fontSize});
  
  void showSettings() {
    print("🎨 Theme: $theme | 🔤 Font Size: $fontSize");
  }
}

// class DarkSettings extends Settings {} // ERROR: tidak bisa diwarisi

void main() {
  final settings = Settings(theme: "Ramadhan Dark", fontSize: 16);
  settings.showSettings();
}

⚙️ Utility Class dengan Static Method

Class final sering digunakan untuk membuat utility class yang berisi method static (fungsi global yang terorganisir).

final class MathUtils {
  static int tambah(int a, int b) => a + b;
  static int kali(int a, int b) => a * b;
}

void main() {
  print("10 + 5 = ${MathUtils.tambah(10, 5)}");
  print("10 × 5 = ${MathUtils.kali(10, 5)}");
}

🎨 Abstraksi (Abstract Class)

Kerangka Dasar Abstract class adalah class yang tidak bisa diinstansiasi langsung, tetapi berfungsi sebagai kerangka bagi class turunan. Method abstrak di dalamnya wajib diimplementasikan oleh subclass.

// Nama  : Putra Dhafir Nashrullah
// NIM   : 251410009
// Kelas : SI2A

abstract class Bentuk {
  double hitungLuas(); // method abstrak (tanpa body)
}

class Persegi extends Bentuk {
  double sisi;
  Persegi(this.sisi);
  
  @override
  double hitungLuas() => sisi * sisi;
}

class Lingkaran extends Bentuk {
  double jariJari;
  Lingkaran(this.jariJari);
  
  @override
  double hitungLuas() => 3.14 * jariJari * jariJari;
}

void main() {
  Bentuk persegi = Persegi(5);
  Bentuk lingkaran = Lingkaran(7);
  
  print("Luas Persegi: ${persegi.hitungLuas()}");
  print("Luas Lingkaran: ${lingkaran.hitungLuas()}");
}

🧩 Mixin dengan State

Code Reuse Mixin adalah cara menambahkan perilaku ke class tanpa pewarisan (mirip multiple inheritance). Dengan with, sebuah class bisa memiliki kemampuan dari banyak mixin. Mixin juga bisa memiliki state (properti).

// Nama  : Putra Dhafir Nashrullah
// NIM   : 251410009
// Kelas : SI2A

mixin CanFly {
  bool _isFlying = false;
  
  void fly() {
    if (!_isFlying) {
      _isFlying = true;
      print("🕊️ Terbang...");
    }
  }
  
  void land() {
    if (_isFlying) {
      _isFlying = false;
      print("🦅 Mendarat");
    }
  }
}

class Bird with CanFly {}

void main() {
  Bird bird = Bird();
  bird.fly();
  bird.land();
}

📝 Kesimpulan

Dari seluruh pembahasan yang telah dilakukan, dapat disimpulkan bahwa:

  • ✅ Sealed class digunakan untuk membatasi pewarisan dalam satu file, cocok untuk state management.
  • ✅ Final class mencegah pewarisan total, ideal untuk utility class.
  • ✅ Abstract class menjadi kerangka dasar dengan method wajib implementasi.
  • ✅ Mixin memungkinkan code reuse tanpa pewarisan tunggal.
  • ✅ Polimorfisme membuat kode lebih fleksibel dan mudah diperluas.
  • ✅ Penggunaan class modifier membuat program lebih terstruktur, aman, dan mudah di-maintain.

Dengan memahami konsep-konsep ini, developer Dart dapat menulis kode yang lebih bersih, ekspresif, dan minim bug.
